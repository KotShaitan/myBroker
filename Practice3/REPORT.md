# Отчет по практике 3: Сравнение стратегий кеширования

## Описание теста
- Приложение: `Practice/Practice3/CacheComparison`
- База данных: PostgreSQL (Docker)
- Кеш: Redis (Docker)
- Набор данных: 1000 записей
- Объем одного прогона: 15000 запросов
- Профили нагрузки:
  - `read-heavy`: 80% чтение / 20% запись
  - `balanced`: 50% чтение / 50% запись
  - `write-heavy`: 20% чтение / 80% запись

## Таблица результатов

| Профиль | Стратегия | Throughput (req/sec) | Средняя задержка (ms/req) | Обращения в БД | Hit rate кеша | Макс. буфер Write-Back |
|---|---|---:|---:|---:|---:|---:|
| read-heavy | Cache-Aside | 309.28 | 3.233 | 6209 | 73.76% | - |
| read-heavy | Write-Through | 334.92 | 2.986 | 3896 | 93.17% | - |
| read-heavy | Write-Back | 341.66 | 2.927 | 3896 | 93.17% | 200 |
| balanced | Cache-Aside | 225.51 | 4.434 | 11487 | 46.85% | - |
| balanced | Write-Through | 262.86 | 3.804 | 7997 | 93.39% | - |
| balanced | Write-Back | 269.03 | 3.717 | 7997 | 93.39% | 200 |
| write-heavy | Cache-Aside | 190.56 | 5.248 | 14426 | 18.52% | - |
| write-heavy | Write-Through | 200.85 | 4.979 | 12099 | 93.61% | - |
| write-heavy | Write-Back | 207.07 | 4.829 | 12099 | 93.61% | 200 |

## Поведение Write-Back при накоплении
- Буфер записи в сценарии `Write-Back` рос до порога `200` элементов, после чего данные сбрасывались в БД.
- Перед финальным `flush` оставалось:
  - `read-heavy`: 82
  - `balanced`: 101
  - `write-heavy`: 101

## Выводы
- Для нагрузки с упором на чтение лучший результат показал `Write-Back` (максимальный throughput и минимальная задержка).
- Для нагрузки с упором на запись также лучший результат у `Write-Back`.
- Для смешанной нагрузки `Write-Back` снова немного быстрее `Write-Through`.
- `Cache-Aside` в этом тесте заметно проигрывает из-за частых промахов кеша и большего числа обращений в БД.
