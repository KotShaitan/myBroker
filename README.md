# 📬 Брокер сообщений на C#

## 📌 Описание проекта

Проект представляет собой простой брокер сообщений, реализованный на языке C#.
Взаимодействие с брокером осуществляется через HTTP API.

Брокер позволяет приложениям обмениваться данными через очереди и топики:
производители (Producer) отправляют сообщения, а потребители (Consumer) получают и обрабатывают их.

---

## 🎯 Цели проекта

* Изучить принципы работы брокеров сообщений
* Реализовать модель Producer / Consumer
* Понять, как работают:

  * очереди (Queue)
  * топики (Pub/Sub)
  * доставка и подтверждение сообщений
* Освоить разработку HTTP API на C#

---

## ⚙️ Основные возможности

* Отправка сообщений через HTTP (Publish)
* Подписка на сообщения (Subscribe)
* Получение сообщений (Pull модель)
* Хранение сообщений в памяти
* Подтверждение обработки (ACK)

---

## 🌐 API (пример)

### 📤 Отправка сообщения

```
POST /publish
```

```json
{
  "topic": "orders",
  "payload": "data"
}
```

---

### 📥 Получение сообщений

```
GET /consume?topic=orders
```

---

### ✅ Подтверждение обработки

```
POST /ack
```

```json
{
  "messageId": "uuid"
}
```

---

## 🧱 Требования

### Функциональные:

* Producer отправляет сообщения через HTTP
* Consumer получает сообщения через HTTP
* Брокер хранит и распределяет сообщения
* Поддержка подтверждения (ACK)

---

### Нефункциональные:

* Асинхронная обработка запросов
* Потокобезопасность
* Простота масштабирования
* Минимальные задержки

---
